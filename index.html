Core Features

Everything you need. Nothing you don't.

Takes replaces dozens of manual steps with a single, intelligent pipeline. Organize, override, switch, render — all from one panel.

🌳

Takes Tree

A unified hierarchy that shows every Scene, View Layer, Group, and Version in one place. Drag, nest, collapse — your entire project at a glance.

⬇️

6-Tier Cascade

Override cameras, worlds, compositors, actions, render presets, and variants at Global, Scene, Group, or individual VL level. Inheritance flows down — overrides stay local.

🔄

Variant Switch

Swap materials, meshes, or visibility states with a single click. Define products, parts, and variant pools — then render every combination automatically.

🎬

Batch Render

Render all View Layers in sequence — foreground or background. The queue applies every cascade override, saves to Smart Output paths, and plays a sound when done.

📁

Smart Output

Token-based output paths that resolve at render time. [scene]/[view_layer]/[camera]_#### — no more manual folder creation.

🎨

Render Presets

Capture your entire render config — engine, samples, resolution, color management — as a reusable preset. Assign via cascade, swap instantly.

👻

Rest State

A neutral fallback pose for every object. When no animation exists on a View Layer, objects return to their "rest" position — no keyframe pollution.

🔍

Inspector Panel

See at a glance which objects are managed, pinned, or dangling. The watchlist catches animation data loss before it happens.

⌨️

Full Hotkey Support

Every tree operation has keyboard shortcuts. Create, duplicate, group, rename, delete — all without leaving the keyboard.

How It Works

From setup to render in three steps

Takes is designed to disappear — set it up once, then let the cascade do the thinking for you.

Organize

Create your Takes Tree — group scenes and view layers into a meaningful hierarchy. Add versions for iterations. Tag with colors for quick visual scanning.

Override

Assign cameras, worlds, actions, render presets, and variant states at any cascade level. Global settings flow down. Local overrides stay local. Zero duplication.

Render

Hit Render. The batch system walks the tree, applies every override, renders each View Layer into its Smart Output path. You get a coffee. It just works.

Why Takes

Built for studios that ship

Takes was designed by a product visualization artist who got tired of the same manual pain — every single project.

⚡

10x Faster Setup

What used to take 30 minutes of manual camera assignment, world overriding, and output path creation now takes 30 seconds with the cascade.

🛡️

Zero Data Loss

The Inspector's dangling action warnings and automatic pinning prevent animation data from ever getting silently destroyed during VL switches.

🧩

Non-Destructive

Pure Blender API. No monkey-patching, no file corruption, no vendor lock-in. Takes works with your existing scenes — just install and organize.

🌍

Background Rendering

Fire off batch renders that run in separate Blender processes. Your main session stays fully interactive. Tree thumbnails update live as results come in.

📐

Blender 5.1 Native

Built on the Slotted Action system. Uses native View Layers, Scenes, and Action Slots — no custom data formats, no compatibility nightmares.

🎯

One Panel, Full Control

Everything lives in the Properties Editor. Takes Tree, cascade overrides, batch render queue, inspector — one tab, zero context switching.
