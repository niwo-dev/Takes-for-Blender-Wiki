# Панель инспектора (Inspector Panel)

**Расположение:** *Properties Editor (Свойства) > Takes > вкладка Inspector*

Панель Инспектора предоставляет список отслеживания (watchlist) для каждого объекта, показывая, какие объекты управляются системой каскада (cascade system), содержит назначенные им действия, слоты и вложенные данные (sub-data: такие как материалы, деревья нодов, ключи формы).

## Управляемые и Закрепленные объекты (Managed vs. Pinned)

Объекты в списке отслеживания находятся в одном из двух состояний:

Управляемые (Managed)
:   Следуют системе каскада и логике дерева. Действие (анимация) объекта автоматически назначается каскадным разрешением активного View Layer.

Закрепленные (Pinned)
:   Исключены из управления каскадом. Объект всегда сохраняет своё собственное активное действие, независимо от переключений View Layer. Чтобы закрепить объект, щелкните на значок **булавки (pin icon)** в его строке.

!!! warning "Открепление (Unpinning)"
    Открепление (Unpinning) объекта, у которого в данный момент есть отдельное активное действие, вызовет диалоговое окно подтверждения, поскольку это действие будет немедленно удалено и заменено базовым действием каскада из-за возврата логики к стандартному процессу управления.

## Список наблюдения (Watchlist)

Список включает информацию по всем объектам, относящимся к текущему и активному View Layer:

| Колонка | Описание |
|--------|-------------|
| **Pin (Булавка)** | Переключает состояние из управляемого (managed) в закрепленное (pinned). |
| **Name (Имя)** | Имя объекта (щелкните по нему, чтобы выделить его во Viewport). |
| **Sub-data icons (Значки подданных)** | Компактные иконки-индикаторы для каждого анимируемого блока данных (материал, нодовое дерево и т. д.). |

### Компактный режим (Compact Mode)

В компактном режиме значки подданных (sub-data icons) отображаются в виде строки миниатюрных индикаторов типов рядом с каждым объектом:

- **Яркий значок** — Этому блоку данным в данный момент назначено действие (есть анимация)
- **Тусклый значок** — В данный момент этому блоку данные не назначено активное действие

При нажатии на любой значок вложенных данных (sub-data icon) откроется всплывающее окно (popover) для управления действиями и слотами на уровне конкретного блока данных (per-datablock).

## Фильтры

Вы можете отфильтровать список наблюдения, используя кнопки в верхней зоне заголовка панели:

- **Фильтр закрепленных (Pinned filter)** — Показать только закрепленные (pinned) объекты
- **Фильтр незакрепленных (Unpinned filter)** — Показать только управляемые (managed) каскадом объекты

Значки фильтров автоматически блокируются и тускнеют (grayed out), когда их значение равно нулю.

## Действия и Слоты (Actions & Slots)

Выберите любой объект из списка отслеживания, чтобы ознакомиться с информацией о его действиях и слотах в нижней части панели инспектора:

- **Список действий (Actions list)** — Показывает все доступные или ссылающиеся на этот объект действия
- **Список слотов (Slots list)** — Выводит все слоты внутри активного на данный момент действия (action)
- **Встроенное переименование (Inline rename)** — Двойной щелчок по имени действия или слота вызывает режим редактирования, позволяя напрямую его переименовать
