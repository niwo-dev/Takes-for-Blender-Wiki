# コンテキストプロパティ (Context Properties)

**場所:** *Properties Editor (プロパティエディター) > Takes タブ > Context (上部トグルより)*

コンテキストプロパティパネルには、現在アクティブになっている View Layer (ビューレイヤー) における「カスケードのオーバーライド（権限上書き）状況」に関する総合的な設定がズラリと表示されます。プロパティの各カテゴリからポップオーバーのメニューにアクセスでき、設定を直接操作することも可能です（※これらはツリー側のカスケードアイコンをクリックした時の動作と同じです）。

## カスケード設定に関するポップオーバー (Cascade Popovers)

Takesツリーの行にあるカスケードアイコンをどれかクリックすると、その専用のポップオーバーメニューが開きます。各々のポップオーバーでは、その「特定の階層レベルのみでの」オーバーライドを、新しく設定したりクリア (解除) したりといった操作が可能です。

### カメラ設定 (Camera Popover)

View Layer ごとに用いるカメラを個別にアサインします：

- **Camera selector (カメラのセレクター)** — 現在のシーンにリンクかつ配置されている有効なカメラオブジェクトのみをドロップダウンメニューに表示して指定します。
- **Camera preset (カメラ用プリセットの適用)** — 保存済みのカメラプリセット (焦点距離、センサーサイズ、被写界深度等) をそのまま適用します。
- **New / Rename (新規作成 / リネーム)** — まったく新しいカメラをここから直接作成したり、現在割り当てられているカメラの名前を変更したりできます。

!!! note "カメラの操作防護 (Camera Guard)"
    ビューポートで選択中のオブジェクトが、そもそもカスケードによって決定（解決）された「正当なアクティブカメラ」では無い場合、カメラのプリセットUIは操作をブロックし無効化されます。
    その場合「カスケード内でカメラを適切に割り当ててください」というインフォメーションメッセージが表示されます。

### ワールド環境設定 (World Popover)

View Layer ごとに用いるワールド環境 (大気/光設定) を個別にアサインします：

- **World selector (ワールド用のセレクター)** — シーン内で利用可能なワールド用データブロックから選びます。
- **World preset (ワールド用プリセットの適用)** — 保存済みのワールドプリセット (HDRI設定等) を適用します。
- **World rule (ワールド指定ルール)** — タグ (Tag) ベースの自動化ルールを使用し、条件に合ったワールドを論理的に自動選択させます。

### アクション設定 (Action Popover)

カスケードによるアニメーションアクションの割り当て状況を表示、および強制管理します：

- **Current action (現在のアクション)** — カスケードの解決結果として、この View Layer に対して現在強固に割り当てられているアクション。
- **Re-apply Cascade (カスケードの再適用)** — もし手動操作でアクションがクリアされてしまったとしても、これを押せばカスケード本来のルールを再計算し、アクション状態を即座に修復・復元します。

### コンポジター設定 (Compositor Popover)

View Layer ごとに個別のコンポジター・ノードツリー処理をアサインします。

### 出力設定 (Output Popover)

レンダリング出力設定の決定と設定を行います：

- **Output rule (出力パスのルール)** — 自動化のためのタグベースによる出力先のパス決定ルール。
- **Render preset (レンダー用プリセット)** — View Layer ごとのレンダリング設定群 (解像度やサンプル数等)。

## 設定権限のオーバーライド解決方式 (Override Resolution)

各プロパティにつけられたオーバーライド設定は、カスケード階層を**「上から下へ（Top-Down）」**と辿ることで解決されていきます。その道中において、"空（未指定）でない値"を最初に見つけた時点で、その設定が勝ち（適用）となります：

```
グローバル(Global) → シーン主組(Scene Group) → シーン(Scene) → VL組(VL Group) → ビューレイヤー(View Layer) → VL改訂版(VL Version)
```

それぞれのアイコン類は、その特定レベル階層にて明確にして明示的な値が設定されている場合は **明るい色 (bright)** で表示され、そうではなく単に親レベルからのカスケード指示を継承して受け取っているだけの場合は、**暗い半透明な色 (dimmed)** で表示されます。
