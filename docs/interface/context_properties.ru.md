# Свойства контекста (Context Properties)

**Расположение:** *Properties Editor (Свойства) > Takes > вкладка Context*

Панель Свойств Контекста отображает настройки переопределения каскада для текущего и активного слоя View Layer. Каждая категория свойств имеет свое собственное всплывающее меню (popover), доступное через значки каскада в дереве.

## Всплывающие меню каскада (Cascade Popovers)

Щелкните любой значок каскада в строке дерева, чтобы открыть связанное с ним всплывающее меню (popover). В каждом окне меню вы можете самостоятельно установить или очистить переопределение именно для этого уровня конкретной иерархии.

### Всплывающее меню камеры (Camera Popover)

Назначить отдельную камеру для любого индивидуального View Layer:

- **Выбор камеры (Camera selector)** — Выпадающий список, показывающий только камеры, жестко привязанные (связанные) к текущей сцене.
- **Пресет камеры (Camera preset)** — Применить или задать ранее сохранённый пресет для камеры (фокусное расстояние, размер сенсора и т.д.).
- **Новая / Переименовать (New / Rename)** — Создать совершенно новую камеру или переименовать её прямо здесь.

!!! note "Защита камеры (Camera Guard)"
    Пользовательский интерфейс (UI) пресетов камеры отключается (замораживается), если выбранный в данный момент объект в базе Blender не является
    именно той "результативной" камерой, с которой официально работает функция текущего каскада. Вы увидите информационное сообщение: «Сначала назначьте камеру в каскаде» (Assign a camera in the cascade).

### Всплывающее меню окружения (World Popover)

Назначить отдельное мировое окружение для вашего View Layer:

- **Выбор окружения (World selector)** — Выберите из доступных блоков данных "World" (Блоки окружения).
- **Пресет окружения (World preset)** — Применить готовый пресет (JSON пресет) окружения.
- **Правило окружения (World rule)** — Выбрать и использовать правило для автоматизированного подбора окружения на базе цветовой идентификации тегов (tag-based rule).

### Всплывающее меню действий (Action Popover)

Просмотр и управление назначениями каскадных действий (анимацией):

- **Текущее действие (Current action)** — Финальное, разрешённое движком каскада действие для этого конкретного слоя View Layer.
- **Повторное применение (Re-apply Cascade)** — Восстанавливает работу действия на уровне каскада после ручного отключения и очистки.

### Всплывающее меню композитора (Compositor Popover)

Назначение специального дерева нодов (node tree) композитора для каждого отдельного View Layer.

### Всплывающее меню вывода (Output Popover)

Здесь вы настраиваете и адаптируете всё, что касается параметров вывода при рендеринге (Output):

- **Правило вывода (Output rule)** — Правило логического формирования пути к файлу на основании тегов (Tag-based output rule).
- **Рендер пресет (Render preset)** — Настройки визуализатора и параметры рендера для каждого View Layer локально.

## Разрешение переопределений (Override Resolution)

Как разрешается и обрабатывается логика переопределения (overrides) сверху вниз в иерархии вашего каскада? Система сканирует её сверху вниз и берет приоритет над самым "первым найденным непустым значением" в структуре дерева:

```
Global → Scene Group → Scene → VL Group → View Layer → VL Version
```

То есть, иконка любого параметра становится **яркой**, когда её значение явно жестко установлено или закреплено непосредственно на её текущем узле / локальном уровне редактирования. Иконка, напротив, становится **тусклой**, когда её реальное значение было унаследовано от узла или уровня, располагающегося выше (уровня родителя - parent level).
