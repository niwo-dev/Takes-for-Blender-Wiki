# Рабочий процесс: Пакетный рендеринг (Batch Rendering)

Этот рабочий процесс детально охватывает настройку и полный запуск пакетного рендеринга по нескольким слоям (View Layers) с автоматическим применением каскадных переопределений и включенной функцией Smart Output.

## Предварительные требования (Prerequisites)

- Рабочая Сцена с несколькими настроенными слоями View Layers.
- Назначенные каскадные переопределения (настроенные камеры, окружения сцен, назначенные пресеты).
- Определенный и активный шаблон имен Умного вывода (Smart Output pattern).

## Настройка Smart Output

### 1. Определение шаблона вывода (Output Pattern)

1. Перейдите во вкладку Свойств (Properties) **> Output**.
2. Включите галочку **Smart Output**.
3. Настройте параметр Шаблон Каталога (Directory Pattern):

    ```
    //renders/[scene]/[view_layer]/
    ```

4. Задайте Шаблон имени файла (File Name Pattern):

    ```
    [view_layer]_[camera]_####.[file_format]
    ```

### 2. Присвоение пресетов рендера (Render Presets)

Для сохранения гарантированно стабильного качества на всех VL:

1. Сделайте все желаемые настройки рендера в меню Blender (движок, сэмплы, разрешения).
2. Сохраните их как Пресет рендера (Render preset), вызвав значок шестеренки (popover) в каскаде.
3. Вы можете закрепить этот пресет сразу на **Глобальном (Global)** уровне, чтобы он повлиял на все VL. Либо привязать его только к определенной **Группе (VL Group)**, чтобы настроить разное качество.

## Запуск Пакетного Процесса

### Режим Foreground (на переднем плане)

1. Откройте боковую панель Takes и саму вкладку дерева.
2. Нажмите на стандартную кнопку **Render** (с иконкой картинки).
3. Очередь немедленно начнет методично обрабатывать каждый VL в строгом порядке, как они стоят в Дереве (сверху-вниз):
    - Сцена и VL переключают контекст.
    - Применят свои каскадные модификации (камеру, мир, анимации, и варианты материалов).
    - Применят активные пресеты рендеринга.
    - Просчитают сцену в движке и сохранят по строгому пути Smart Output.
4. Вы будете наблюдать прогресс и статус "выполнения" для каждого отдельного VL в графическом окне очереди.

### Режим Background (фоновый режим)

1. Нажмите на кнопку **Рабочего стола Desktop** в панели навигации вместо кнопки рендера.
2. Интерфейс Blender'а останется полностью разблокирован для вас и отзывчив, пока все фоновые процессы рендерят финальную картинку незаметно для системы.
3. Маленькие встроенные миниатюры слоев в вашем Дереве будут послушно обновляться по мере завершения очередного VL.

## Мониторинг Прогресса

Боковая панель для отслеживания рендера показывает:

- Полосу визуального прогресса на каждый VL.
- Текстовый индикатор статуса (В очереди/Pending, Рендерится/Rendering, Сохраняется/Saving, Завершено/Done, Ошибка/Failed, Отменено/Cancelled).
- Для слоев, выдавших статусы "Ошибка (Failed)", достаточно навести курсор мышки: выведено будет всплывающее окошко (tooltip) с причиной сбоя.

## Структура вывода файлов

Оперируя заданными чуть выше шаблонами структуры ([view_layer]... и т.д.), ваша финальная папка на диске будет выглядеть примерно вот так:

```
renders/
├── Kitchen/
│   ├── Front_3-4/
│   │   ├── Front_3-4_CamHero_0001.png
│   │   └── Front_3-4_CamHero_0250.png
│   └── Top_Down/
│       ├── Top_Down_CamTop_0001.png
│       └── Top_Down_CamTop_0250.png
└── Bathroom/
    └── ...
```

## Дополнительные советы

!!! tip "Звук окончания рендера"
    Вы можете задать свой кастомный или системный звук во вкладке **Preferences > Features** (В общих настройках самого аддона).
    Когда фоновый режим (Background renders) закончит все свои задачи, аддон заботливо позовет вас! Инструмент поддерживает любые системные звуки, а также кастомные форматы `.wav`.

!!! tip "Предварительный просмотр (Preview)"
    С помощью сочетания ++alt++-левый клик (alt-click) на иконке рендера конкретного VL вы сможете просмотреть самый последний отрендереный выходной графический файл без необходимости рендерить его повторно.

!!! warning "Сохраняйте заранее (Save First)"
    Фоновый рендер (Background) категорически требует, чтобы ваш рабочий файл `.blend` был предварительно сохранён на диск (т.к запускается независимая копия Blender). Рендер на переднем плане (Foreground) полностью работает на имеющемся сейчас открытом состоянии оперативной памяти (memory state).
