# 常见疑问及解答汇总 (FAQ)

## 全局通法类门

??? question "咱要用到啥底子年代段位的 Blender 才行？"
    Takes for Blender 死线门门槛要求是 **Blender 5.1** 或着那些个比他更新晚生的小弟后代都可以。因为这插件的生父基因决定它全命维寄托挂带依仗一门首出在 Blender 5.0 版上的那门绝技叫 Slotted Action 系统去运做。

??? question "那 EEVEE 跟那高冷 Cycles 他全对付得了通吃吗？"
    没跑的！整个它下头带的什么大瀑布流水线 (cascade) 跟那些个去打黑工的批量出片系都是完全“吃百家饭和随大流(engine-agnostic)” 根本不管底下是谁，它就干事。连给你收存的那本绝世“神算参数秘典”（Render presets）都能自带超强眼力全顺自动看好并带上那专用大出图主脑自己的小九九和绝密特性指标打包！

??? question "这爷们儿跟其他别家大流牌子插件混坐一堆挤一块跑时不会火星对撞出命吗？"
    老天保准绝不！咱们这款叫 Takes for Blender 只玩正统阳关大路也就是它全遵循用最最标准清白地老实人 Blender 内设自流门大准标准 (standard Blender API) ! 绝不做任何啥叫走歪门底拉后改主线脉 (monkey-patch) 去变其元神的手足段俩。如果若你那天真的不幸看它两互砍起火碰撞了报错，烦劳发帖或者电告知上禀。

## 是关于他那带门面 Takes 体系和那所谓的大瀑布式灌注(Cascade)之事

??? question "咱只是随手那么一点给切换了下一个别的 View Layers 时，那我家那些被做下动作轨道的玩意(animation)究竟怎生被处置发落了？"
    被造有动作且随行绑有东西的主子如若跑去了别的完全没受委派该干嘛的层时（即这 target 没受接领派动作档案)，那此物件会立马跟脱骨还乡回胎一样秒切回缩至到出厂打标定位之躯！这也就是它的静止出基位（Rest State）。这规矩雷打不动！假如真在别台它有特权领有自家的派发的出令动作码！那这级联机(cascade action)会绝无卡断将它强制换身替衣接盘执行起此位专享档的行动走起来！

??? question "咋地我这就看俺那卡图标面上它是给显示个没啥生气的灰黯哑色调呢 (dimmed)？"
    你放心那没坏没毛病！凡挂上这 "半明半显的哑光脸壳" 就是对你视觉传递告诉你： 此物本层的这一套指示，乃只是个**老老实实随传依下从其祖宗/或大组（inherited)** 那里去乖接管白拿承下继承权用罢了！
    反倒当他如吃满灯珠泛出个红红绿满发光夺彩的大亮高闪相(Bright icon)之时，这就代表特等大特权此层就是 **不甩你！被单强制划了线拍在重权位(explicitly set)** 立规矩压你！
    手抖按着那个 Alt 加左鼠标轻叩即能把他清平夺权重回附庸听话继承从属！

??? question "能给我这单拉的某一位 View Layer 取个特裁有它私密长短专用的帧轨道刻线区间段(Frame ranges) 吗？"
    非也非也不能及也老铁！整个做帧头截尾的时长划定是在人家大院 Blender 里那可谓是最高顶上一层的权座。去不同的大院落 "Scenes(全场分支全场景)" 这个超大型系统界域的树枝那另起去弄那些不同时局吧大段吧。不能为这底下小子(View Layer)开破局的。

## 这是关于底下管收察房大官人 (Inspector)之事

??? question "咋这这列上猛不丁都跟火烧出血似的标上刺目艳红印子(red)了？"
    这是极其紧急的大灾难天罗地网预警：这种刺辣辣大底色印染在整个待查巡检栏的 (watchlist)，这是大挂特唤一个叫作什么 **挂死没出处幽灵游荡大孤儿（dangling action warning）** 的极其恐怖前兆令预警了 — 他就是说：这底头那只特定的大物有且确实揣有一条什么没名身份或者不隶属于这大瀑布 (cascade)管下名门管着它那带特大动画包裹，而且它还要到处乱逛乱去跳级切换。你不硬给它上个大枷锁固定 **拴大铁链(Pin)** 把这爷送了神，就是老老实实给其打开让他被老天包着统走挂自动跟接归名**接受全招安(Auto-Assign)** 的二选一选项，否则那它这就等着死后消亡散魄没处找去。

??? question "啥是个叫被统管招安了(Managed)又何谓是一辈子拴这死铁锁大刑伺候锁(Pinned)啦？"
    被招安入流纳入王法的就是那听着天爷那大连排派指令（cascade）行规法度的乖角儿（**Managed**）：走过层时其自身动作身由不由已随该层大指派换来变去下着。而所谓打了铁桩的死刑户（**Pinned**）那帮：简直是无法无天天字帮号。它们是拿命把其那个所拥有身动作死绑硬夹不管狂风乱作改层跨台换啥了哪怕天崩，他仍然死死持此不动。想要其屈服受刑你仅须轻触该行列上面去敲点上那个带刺的大图钉印码(pin icon)便当即上了绑枷。

## 那管排房排号全出料的后生系统事 (Batch Render)

??? question "我点了咋这破算图程序(render)不给我开转哪？"
    你老先别气自己查查这单点是否落网了可好不：
    1. 你存了底档名这正主底件没(没点存档绝对不能使唤你那隐后排那不让瞅打工人的 background mode)。
    2. 你在这层挂上随便啥用来引光接影的眼摄机机位归级头等事(cascade)了吧？
    3. 全机器是不是抽经全卡死死了(别怕把键盘按死了加着 Alt 加鼠标暴左猛击那个排房小相机键按钮(Renderbutton)强灌重灌洗全大局救活他)。

??? question "能不能只对准我这有兴头专去挑的那某家某个特定的单只 View Layers 放去开算(render)?"
    必须可行啊大兄弟。你就把那树底层留好的那个排排点的小钩选钮 (multi-select)放开了去激活！你再去选中哪些要它出来的兵点勾打底后。你拍大照片那印子钮 Render。此号排单上便专且只装那些打好印头标符被送命队列名单上的那批家伙才发过去! 

??? question "出大货这老多图全排都统塞搁哪去了？"
    一切之安排与去宿均悉听老人家之言、此所谓之老人家神机妙术口占的 [神算智能老者输出路线 (Smart Output pattern)](smart_output.md)给包圆了定道。
    你翻翻那个名为 **大原家总盘产出之法栏 Properties > Output** 以亲去复目对数它手现正把弄何局阵版盘型制。
    在真正将要把那单个兵发放到炉口算前绝底的那万急生死关那最后绝发时辰这层内法力变量才会算齐了定去名与字了最后那死名牌!

## 这些是关那套皮肤款式魔术戏法之细事 (Variant Switch)

??? question "我要是在这又能让他给全全替代死拼又教他入到那拿牌子按编排套底去配着混用混编做连法可以吗?(Swap 拼和拿 Pool混杂) "
    我跟您交代一句底话，这一只所谓的名号 "组件分支（即一个单一的Part）"一次生死周期里只能一回仅专从只走吃某一道派系法则体系模式来管死不得多法共出。但是！别忘了！在一大家子全头总把子也就是 "那个全大宗门系大总线(Product)"所同统帅辖下之互为独立外戚那些千千万外亲别系之组件旁支那帮（Different Parts within the same Product）他们私底下各自统门是全都不互咬并相安和各练各得大法相处是无隔碍能同时兼容做工全跑并发开机滴! 

## 收查毛病开方子治病的对症查收卷案集 (Troubleshooting)

??? question "这简直鬼扯！弄好装完了根本就一点半点痕迹也没看到有上这表来(The addon isn't showing up )。"
    大能尊者再好生且劳驾看看你自己到底是拿的什么法去传道装此真名仙家正器的。你莫去用解压破拆！唯遵真言: 直从其正根正形最圆全原本未拆封带这那纯 `.zip` 后果圆包装态给直接让其入内服送至装点(install)便可出奇效。
    另者也开下您家里这 Blender 大盘总台瞧一眼那有无那些脏眼打码红字报（进 Window > 把它那总控制系统密保柜开关出示点落 Toggle System Console）抓不全毛病抓着有误的没。

??? question "我在那就平平常去推那些景视图拉(View Layer switches)那家伙次次给我脸爆卡停掉链当机啦(crashes)。"
    一般那凡出这等祸事这通常就是你里边掺进乱吃进和那另外什么旁的别姓他家那没安分的别个系统套件那系统里面什么带底链听诊的线搭错了(conflict with another addon's depsgraph handler)跟我的起冲打架了罢。
    建议: 先单吊这几家闲杂等下黑给掐掉以查准了看到底是在谁那出毒素并单独给剔他出来把。顺带去我们的总台后台大管勾上那个叫专门查鬼号子的 Debug logging(把这个一开), 到时再将所抄下来的全宗手稿日记抄底拍那把拿来直接做病案指控单呈我们看!
