# レンダープリセット (Render Presets)

**レンダープリセット (Render Presets)** とは、Blender の各種レンダリング設定群を JSON ベースの超軽量なスナップショットとして抽出したものです。これらのプリセット群は、カスケードシステムによって View Layer ごとに柔軟に割り当てることが可能です。

## プリセットの取り扱いタイプ (Preset Types)

| タイプ | それが何を内包し記録してキャプチャするか |
|------|-----------------|
| **Render (レンダー)** | レンダリングエンジン種、サンプリング数、各種解像度構成、カラーマネジメント設定、及び Film（フィルム）設定関連。 |
| **Camera (カメラ)** | 焦点距離、センサー設定サイズ、被写界深度 (DOF)、クリップ境界距離。 |
| **World (ワールド)** | 環境・グローバルライティング設定、背景の定位置カラー、AO (アンビエントオクルージョン) 設定。 |
| **File Output (ファイル出力)** | 出力先フォーマット型、出力色深度定義、画像圧縮率、さらには Smart Output (スマート出力) 側の指定トークンパス。 |

## プリセットの作成方法 (Creating Presets)

1. まずは通常通り、Blender の Properties Editor (プロパティエディター) から、お目当てのレンダリング設定群を好きなように構築・設定します。
2. Takes ツリーの該当のカスケードアイコン行にある **歯車アイコン (gear icon)** をクリックして、プリセット用のポップオーバー・メニューを開きます。
3. **Save as Preset (プリセットとして保存)** をクリックして、分かりやすい名前を付けます。
4. この設定データは、システムのユーザー・プリセットディレクトリ内に軽量な一連の「JSON ファイル」として強固に保存され定着します。

## プリセットの適用・命令割り当て (Assigning Presets)

各種のプリセットは、カスケード群を通して指示・割り当てが行われます。つまり、これらは「以下のあらゆる階層 (Tier)」のどこからでも命令・セットすることが可能です：

- **Global (グローバル階層)** — 大元の全プロジェクトにかかる「絶対たるデフォルトのレンダリング設定ベース」として指定。
- **Scene Group (主組階層)** — 固有の特定のシーン群だけで「共有使回し」する設定として指定。
- **View Layer (VL本体層)** — 各テイクのショット1枚ごとの徹底的な構成指定。
- **VL Version (VL改訂版層)** — バージョン内のみで独自に調整反映させたい個別設定。

## プロパティ値の「汚れ」状態検知 (Dirty State)

一旦プリセットが適用された「後」に、あなたが Blender のレンダリング設定を後から手動で変更 (いじって改変) した場合、システムはそれを即座に察知して、設定が「汚れ (Dirty)」状態になったことを示す **警告インジケーター** を表示します：

- **Accept (受け入れ・受諾)** — 現在の手動での「汚れ」変更分を正しいものとして受け入れて、元の大本プリセットである自身の JSON ファイル本体へと「上書き保存」します。
- **Revert (破棄と修復)** — 手元のその勝手な変更分を完全に破棄し、親であるプリセットが持っていた本来の「清らかな」設定値によって強制修復・復元します。
- **Accept & Sync (++alt++ + Accept クリック)** — 手元の変更結果を親プリセットとして上書き保存（受諾）した上で、さらにこのプリセットを使用している **「階層チェーン上のすべてのVLやScene」にまで、即時一括でこの変更による強制同期を伝播**・波及させます。

!!! note "階層出所の明記・インジケーター (Tier Indicators)"
    各種のプリセット・ドロップダウン用メニューでは、
    組み込みのプリセットなのかユーザーの自作による設定なのかを素早く区別して見分けるために、
    その後ろに `[Tier]` つまり『出処元・接尾辞』 (例：`[Addon]`, `[User]`) が明記表示されます。
