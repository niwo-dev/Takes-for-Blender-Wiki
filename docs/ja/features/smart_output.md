# スマート出力 (Smart Output)

**スマート出力 (Smart Output)** つまり「自動パス解決」システムは、レンダリング時に要求される「出力先のパス（宛先）」を、動的なトークン (代入文字) 手法で解決提供する強力なシステムです。一枚一枚のレンダリング結果出力に対して手動で名前を付ける苦痛の代わりに、「コンテキストに沿って賢く拾い上げられた固有変数化文字列たち」が、出力用の・魔法の代用トークンとして完全に置き換えられ展開します。

## Smart Output (スマート出力) の有効化手順

1. Blenderの Properties Editor (プロパティエディター) から、**Output (出力)** パネルの項目枠を開きます。
2. そこに居る **Smart Output** のトグルスイッチを オン (有効化) にします。
3. 現れた **Directory Pattern (主力出力ディレクトリの書式設定)** と **File Name Pattern (名付け用パターンの書式設定)** の項目に、トークン変数代用文字列をセットします。

## トークンの書式構文 (Token Syntax)

それぞれのトークン (変数代入文字列) は、システム設定された「括弧包み文字 (brackets)」によって厳重に囲まれています。デフォルトの括弧スタイルでは、四角い角括弧 (`[ ]`) が使用されます：

```
[view_layer]_[camera]_####.[file_format]
```

上記の変数トークン列が入力されると、システム実行時に、実際の出力名として以下のような文字列群へと見事に解決置換されて保存されます：
(例： `Front_3-4_CamHero_0001.png` )

### 多彩なブラケットの許容スタイル (Bracket Styles)

**Addon Preferences (アドオンプリファレンス設定) > Behavior Options (挙動設定) > Syntax Brackets (構文用ブラケット)** の項目から、あなたの好みに合わせたお好きなパターンの「囲い文字」スタイルを選択・指定することができます：

| 選べるスタイル | 構文の記述 | 使用展開例 |
|-------|--------|---------|
| Square (角括弧型) | `[token]` | `[view_layer]_####` |
| Curly (波括弧型) | `{token}` | `{view_layer}_####` |
| Angle (アングル型) | `<token>` | `<view_layer>_####` |
| Parens (丸括弧型) | `(token)` | `(view_layer)_####` |
| Percent (パーセント型) | `%token%` | `%view_layer%_####` |
| Dollar (ドルマーク型) | `$token$` | `$view_layer$_####` |
| Hash (ハッシュ連番型) | `#token#` | `#view_layer#_####` |

## ご利用可能な全トークン一覧 (Available Tokens)

### 状態や階層関係からの抽出 (Context Tokens)

| 実装トークン | 何に対して解決置換されるのか | 出力される例 |
|-------|-------------|---------|
| `scene` | 実際に稼働中の アクティブ・シーンの名前 | `Kitchen` |
| `view_layer` | 実際に稼働中の アクティブ・ View Layerの名前 | `Front_3-4` |
| `camera` | その時実際に運用されている カメラーの名前 | `CamHero` |
| `group` | そのVLが属している VL Group (小組) の名前 | `Hero_Shots` |

### アニメーション情報からの抽出 (Animation Tokens)

| 実装トークン | 何に対して解決置換されるのか | 出力される例 |
|-------|-------------|---------|
| `frame` | その際現在の カレントフレームの番号位置 | `42` |
| `frame_start` | 全シーンベースでの開始フレーム指定 | `1` |
| `frame_end` | 全シーンベースでの最終結末の終了フレーム指定 | `250` |
| `frame_range` | 開始-終了の連番・結合範囲 | `1-250` |
| `action` | 本来のアクティブな元シーン側アクション指定 | `CameraFly` |
| `tks_action` | 強力なカスケードが"上書きし直した" 解決後のアクション指定 | `SpinX` |

### レンダリング情報からの抽出 (Render Tokens)

| 実装トークン | 何に対して解決置換されるのか | 出力される例 |
|-------|-------------|---------|
| `engine` | 採用された レンダリングエンジンの呼称名 | `cycles` |
| `samples` | 打たれた サンプル数 (Sample count) | `128` |
| `res_x` | X側への 画質レンダリング解像度サイズ | `1920` |
| `res_y` | Y側への 画質レンダリング解像度サイズ | `1080` |
| `file_format` | エンコード指定された・書き出されるファイルの接尾フォーマット | `png` |

### システム基本情報からの抽出 (System Tokens)

| 実装トークン | 何に対して解決置換されるのか | 出力される例 |
|-------|-------------|---------|
| `blend` | 今現在開いて作業している `.blend` ファイルの大本名 | `project` |
| `date` | 現在の・マシン側出力システム打刻日付 | `2026-03-29` |
| `time` | 現在の・マシン側出力システム打刻時刻 | `14-30-00` |
| `sep` | 間を断ち切る 区切り文字 (Separator character) | `_` |

!!! tip "心強いチートシートの呼び出し機能"
    Output（出力）パネルのヘッダーにある **Syntax Tokens (構文トークン：本の小さなアイコン)** 버튼を選択してクリックしてみてください ──
    全カテゴリーにまたがった利用可能全トークンを包括し、対話型でパッと引ける「完全なトークン一覧リファレンス」が即座に呼び出されます。

## 区切り文字・断絶字 (Separators)

`[sep1]` から `[sep5]` までに割り振られた、最大 5つ の「セパレーター(断絶)トークン」を個別に呼び出して、ファイル名上での名前を断絶・区切るために活用できます：

- `[sep1]` = `_` フォルダ階層・並びに変数用
- `[sep2]` = `-` 多重変数用のマイナス記号
- `[sep3]` = `.` バージョニングを付与するためのドット区切り用

これらの、セパレーター (separator characters) 各々がどんな記号を出力するかについては、**Addon Preferences (アドオンプリファレンス)** から各人５種とも柔軟に指定・上書き設定変更できます。

## フレームの連番ナンバリング (Frame Numbers)

パディング（桁詰め）されたフレームの連続番号を出力したい場合は、Blenderに古くからネイティブに備わっている `####`（ハッシュ4つ）の独自指定構文スタイルを単純にそのまま使用してください。Smart Output システムはこれらが入力に混信した際も完全にその意味を汲み取って保持し、現在の打刻フレーム番号の状況と共に即座にパディング化で置換処理を行い、その結果を素早くライブプレビュー・表示させます。
