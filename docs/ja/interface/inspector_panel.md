# インスペクターパネル (Inspector Panel)

**場所:** *Properties Editor (プロパティエディター) > Takes タブ > Inspector (上部トグルより)*

インスペクターパネルは「どのオブジェクトがカスケードシステムによって自動的に管理されているか」、またそれぞれの割り当てられたアクション、スロット情報、および各種のサブデータ（サブ化されたマテリアル、ノードツリー、シェイプキーなど）を監視するための「オブジェクトごとのウォッチリスト（監視リスト）」を提供します。

## 対象の管理状況：「管理状態」と「ピン留め状態」 

ウォッチリストに表示される全てのオブジェクトは、以下のふたつの状態のいずれかになっています：

管理状態 (Managed)
:   システムのカスケードの力に従っている状態。つまりそのオブジェクトの「アニメーション (アクション)」は、その時にアクティブな View Layer のカスケード司令によって**完全に自動的に割り当て（上書き）**られます。

ピン留め状態 (Pinned)
:   システムへのカスケード管理から自らを「免除」している状態。View Layer をどれだけ切り替えても、そのオブジェクトは**現在自分が持っているアクションを頑なに保持し続けます**。行の右端にある **ピン留めアイコン (pin icon)** をクリックすることで、オブジェクトをこの状態に変えることができます。

!!! warning "ピン留めの解除 (Unpinning)"
    独自のアクションを持っているオブジェクトのピン留めを不用意に解除しようとすると、確認ダイアログが表示されます。
    なぜなら、解除した瞬間にあなたが作成したそのアクションは剥がされ、元のカスケード司令によるアクションへと強制的に上書き (置換) されてしまうからです。

## ウォッチリスト大枠 (Watchlist)

ウォッチリストには、現在の View Layer との関連があるすべてのオブジェクトのみが抽出され表示されます：

| カラム | 説明 |
|--------|-------------|
| **Pin (ピン留め)** | 管理状態 / ピン留め状態 を切り替えるトグルボタン。 |
| **Name (名前)** | 対象のオブジェクト名。(ここをクリックするとビューポート内でも直ちに選択されます)。 |
| **Sub-data icons (サブデータアイコン群)** | 一つ一つのアニメーション化された各種データブロック (そのマテリアルやノードツリーなど) のためのコンパクトなアイコン群。 |

### コンパクトモード (Compact Mode)

コンパクトモードでは、オブジェクトの横に、そのタイプごとのサブデータアイコンとして並んで表示されます：

- **明るい単色アイコン** — その対象のデータブロックには間違いなくアクション（アニメーション）が割り当てられています。
- **暗くくすんだアイコン** — そのデータブロック自身には何のアクションも割り当ててられていません。

いずれかのサブデータアイコンをクリックすると、そのデータブロックの奥深くへと干渉するための「アクションおよびスロット管理専用のポップオーバー」が開きます。

## フィルター活用 (Filters)

リストの上部ヘッダーにある各種ボタンを使って、ウォッチリストの表示をフィルタリング（絞り込み）できます：

- **Pinned filter (ピン留め専用フィルター)** — 「ピン留め状態」であるオブジェクトのみを抽出します。
- **Unpinned filter (管理下専用フィルター)** — 「システム管理状態」であるオブジェクトのみを抽出します。

該当する要素のカウントがゼロである場合、そのフィルターアイコンは無効化（グレーアウト）されクリックできなくなります。

## アクション と スロット (Actions & Slots)

ウォッチリスト内でいずれかのオブジェクトを選択すると、インスペクターパネルの下側セクションに、そのオブジェクトが持つアクションとスロットの全詳細がリストとして開示されます：

- **Actions list (アクションリスト)** — そのオブジェクトを参照源としている全てのアクション歴史を列挙します。
- **Slots list (スロットリスト)** — 現在アクティブなアクション内を構成している全てのデータスロットを列挙します。
- **Inline rename (インラインリネーム)** — アクション名またはスロット名をダブルクリックすることで、その場で名前を直接変更できます。
