## Capacidades Principales

| Característica | Descripción |
|---------|-------------|
| **Takes Tree** | Jerarquía unificada: Scene Groups → Scenes → VL Groups → View Layers → Versions. |
| **6-Tier Cascade** | Resolución de anulaciones (overrides) desde Global hasta el nivel de Version para cámaras, mundos, acciones y más. |
| **Variant Switch** | Sistema de variantes de producto con intercambio de materiales por contexto, inyección de preajustes y gestión de pools. |
| **Rest State** | Sistema de acción de referencia con reflejo automático que conserva los fotogramas clave intactos a través de las animaciones. |
| **Batch Render** | Motor de renderizado modal (en primer o segundo plano) con un sistema de tokens Smart Output para nombrar archivos. |
| **Render Presets** | Preajustes basados en JSON para renderizado, cámara, mundo y configuración de salida — por View Layer. |
| **Inline Previews** | Miniaturas del viewport en vivo junto a cada View Layer en el árbol. |

!!! tip "Versión de Blender"
    Takes for Blender requiere **Blender 5.1** o superior. Utiliza el sistema Slotted Action introducido en Blender 5.0.
