# Дерево вариантов (Variant Tree)

**Расположение:** *Properties Editor (Свойства) > Takes > вкладка Variant Switch*

Дерево вариантов (Variant Tree) управляет всеми вариантами продукта — будь то различные комбинации материалов, цветовые решения, либо активные/неактивные состояния продукта. Для этого используется иерархическая структура из: Продуктов (Products), Состояний (States) и Деталей (Parts).

## Иерархия (Hierarchy)

```mermaid
graph TD
    P[Product (Продукт)] --> S1[State A (Состояние 1)]
    P --> S2[State B (Состояние 2)]
    S1 --> PA[Part: Body (Деталь: Корпус)]
    S1 --> PB[Part: Cap (Деталь: Крышка)]
    S2 --> PC[Part: Body (Деталь: Корпус)]
    S2 --> PD[Part: Cap (Деталь: Крышка)]
    PA --> M1[Material Pool (Пул материалов)]
    PB --> M2[Material Pool (Пул материалов)]
```

Продукт (Product)
:   Элемент и контейнер верхнего уровня (например, "Бутылка", "Часы").

Состояние (State)
:   Названный вариант продукта (например, "Золото", "Серебро", "Матовый черный").

Деталь (Part)
:   Локальный компонент, который конструктивно привязан к конкретной коллекции в Blender (например, "Корпус", "Крышка", "Ремешок").
    У каждой детали (Part) имеется собственный пул материалов со слотами под соответствующими индексами.

## Использование

### Переключение вариантов (Switching Variants)

Просто щелкните **значок ромбика (diamond icon)** рядом с любым неактивным состоянием, чтобы мгновенно предпросмотреть этот визуальный вариант во Viewport. Текущее активное состояние визуально отображается в виде закрашенного круга.

### Пул материалов (Material Pool)

Каждая деталь имеет пул материалов (Material pool) — список всех материалов, которые можно менять между собой:

1. Назначьте свой материал в первый пустой слот (новый слот создается потом автоматически сам по себе).
2. Используйте **индекс пула (pool index)**, чтобы определить и выбрать активный материал для конкретной детали.
3. При переключении между различными состояниями система заменяет материалы в соответствии с индексом в пуле материалов у каждого состояния.

### Назначение коллекций (Collection Assignment)

Каждую Деталь (Part) необходимо жестко привязывать к отдельной коллекции Blender. Абсолютно все объекты, находящиеся как внутри этой коллекции, так и внутри дочерних подколлекций (child collections), будут реагировать на замену их материала.

## Теги вариантов (Variant Tags)

Любое из Состояний можно опционально отметить так называемыми "Вариант-тегами" (**Variant tags**) для базовой организации и последующих манипуляций процессом Smart Output, подставляя значение в токен автоматического разрешения имени файла `{variant_tag}`.
