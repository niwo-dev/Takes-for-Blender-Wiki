# Рабочий процесс: Варианты анимации (Animation Variants)

Этот рабочий процесс демонстрирует, как безболезненно настроить совершенно разные анимационные дубли (takes) для каждого отдельного View Layer — например, продукт безупречно вращается на одном слое, наклоняется на втором и взрывается на третьем. И всё это работает со своими независимыми камерами.

## Подготовка сцены

### 1. Создание слоев (View Layers)

Создайте по одному слою (View Layer) для каждого состояния анимации:

1. Откройте панель Дерева снимков (Takes Tree).
2. Нажмите **+** → **Add View Layer** для каждой нужной вам анимации.
3. Обязательно дайте им описательные имена: "Spin (Вращение)", "Tilt (Наклон)", "Explode (Взрыв)".

### 2. Сгруппируйте их

1. Выберите первый VL (View Layer).
2. Нажмите ++ctrl+g++, чтобы немедленно создать группу и назовите её "Анимации".
3. Аккуратно перетащите туда остальные VL.

### 3. Назначьте камеры

Для каждой анимационной сцены, вероятно, потребуется свой собственный ракурс камеры:

1. Щелкните значок камеры в строке каждого конкретного VL.
2. Назначьте соответствующую камеру из списка.

## Анимирование

### 4. Анимация каждого слоя отдельно (Per View Layer)

1. Переключитесь на View Layer слоя "Spin", щелкнув по нему мышью в дереве.
2. Система Каскада (Cascade) немедленно и автоматически назначит для этого слоя выделенный блок анимации (action).
3. Анимируйте свои объекты. Все созданные ключевые кадры (Keyframes) надежно сохраняются в привязанном только к этому конкретному VL действии.
4. Переключитесь на "Tilt" — объекты автоматически "отщелкнутся" (snapped) назад в свои нейтральные базовые позы, благодаря Системе Базового Состояния (Rest State).
5. Теперь создайте совершенно другую анимацию исключительно на этом VL.

!!! tip "Состояние Покоя (Rest State)"
    Система Rest State бдительно следит над тем, чтобы объекты послушно
    возвращались в свою изначальную позу при переключении на неанимированный
    View Layer. Всё автоматизировано и происходит без вашего участия.

### 5. Рендеринг всех анимаций скопом

1. Активируйте режим мульти-выбора (multi-select) в нижней части панели дерева.
2. Отметьте галочками все View Layers с вашими анимациями.
3. Нажмите кнопку **Render**.
4. Умный Вывод (Smart Output) безошибочно назовет каждый выходной файл, автоматически используя названия самих VL.

## Результат

Теперь каждый View Layer безупречно имеет:

- Свой собственный, уникальный ракурс камеры.
- Собственное анимационное действие (action).
- Собственное имя для выходного рендеринг-файла.
- И всё это полностью и централизованно управляется из одного унифицированного дерева.
