# Снимки (Takes)

**Take** — это именованная группа View Layers (слоев просмотра), которая представляет конкретную конфигурацию вашей сцены — ракурс камеры, настройки освещения, вариант материала или состояние анимации.

## Концепция

В кино и фотографии «дубль» (take) — это одна версия кадра. Takes for Blender расширяет эту концепцию до системы View Layer в Blender, предоставляя вам:

- **Независимые назначения камер** для каждого View Layer
- **Независимое мировое окружение (World environments)** для каждого View Layer
- **Независимые деревья нодов композитора** для каждого View Layer
- **Независимые пресеты рендеринга** для каждого View Layer
- **Независимые действия анимации (actions)** для каждого View Layer

## Организация

Takes организованы в иерархическом порядке внутри дерева (Takes Tree):

| Уровень | Назначение | Пример |
|-------|---------|---------|
| **Группа Сцен (Scene Group)** | Организация верхнего уровня | "Интерьер", "Экстерьер" |
| **Сцена (Scene)** | Сцена Blender | "Кухня", "Ванная" |
| **Группа VL (VL Group)** | Логическая группировка VL-ов | "Hero Shots", "Детали" |
| **Слой (View Layer)** | Фактическая единица рендера | "Фронт 3/4", "Вид сверху" |
| **Версия VL (VL Version)** | Именованный снимок настроек VL | "v1 теплый", "v2 холодный" |

## Создание Снимков (Takes)

### Добавление View Layer

1. Нажмите **+** на боковой панели дерева.
2. Выберите **Add View Layer**.
3. Новый VL унаследует свойства от активного на данный момент View Layer.

### Группировка View Layers

1. Выберите View Layer в дереве.
2. Нажмите ++ctrl+g++, чтобы создать Группу (VL Group), содержащую его.
3. Перетащите в эту группу другие View Layers.

### Версии VL

Создавайте именованные "моментальные снимки" каскадных настроек конкретного View Layer:

1. Выберите View Layer.
2. Нажмите **+** → **Add Version** (Добавить Версию).
3. Каждая версия сохраняет собственные переопределения (overrides) для камеры, окружения, анимационных действий и пресетов.

!!! tip "Быстрое переключение"
    Переключайтесь между Версиями VL для мгновенного сравнения различных 
    конфигураций без необходимости дублировать целые View Layers.
