# Пресеты Рендера (Render Presets)

**Render Presets** — это снимки состояния (snapshots) настроек рендера Blender, сохраненные в формате JSON, которые в любой момент могут быть назначены любому View Layer индивидуально посредством системы каскада (cascade).

## Типы Пресетов

| Тип | Что он сохраняет/захватывает |
|------|-----------------|
| **Render (Рендер)** | Выбор движка, кол-во семплов, размер/разрешение кадров, цветокоррекция, настройки пленки (film). |
| **Camera (Камера)** | Фокусное расстояние, размеры физического сенсора, глубина резкости (DOF), и дистанции клиппинга (clip). |
| **World (Мировое Окружение)** | Данные окружающего освещения, основной цвет фона, настройки окклюзии пространства (AO settings). |
| **File Output (Вывод файлов)** | Формат, тип расширения файла (jpg/png/exr), глубина цвета, сжатие и пути/правила для Smart Output. |

## Создание Пресетов

1. Задайте нужные вам параметры рендера, камеры или окружения в основном меню Редактора Свойств Blender (Properties Editor).
2. Вызовите всплывающее меню (popover) пресетов у любого слоя (кликнув на значок шестеренки в каскаде).
3. Нажмите кнопку **Save as Preset** и дайте ему читабельное имя.
4. Вы только что сделали пресет, и он успешно сохранился как отдельный JSON файл прямиком в вашей пользовательской директории.

## Назначение Пресетов

Интеграция или назначение пресетов идёт в тесном контакте с каскадной системой — вы имеете возможность назначить пресет на любой уровень структуры:

- **Глобальный уровень (Global)** — Стандартные настройки для всего открытого проекта.
- **Группа Сцен (Scene Group)** — Общие, объединенные параметры для определённой группы сцен.
- **Слой (View Layer)** — Персональные настройки конфигурации для отдельных шотов/кадров.
- **Версия VL (VL Version)** — Специфические поправки только в рамках данной версии.

## "Грязное Состояние" (Dirty State)

В случае если настройки визуализации претерпели сторонние правки после загрузки пресета в сцену, интерфейс укажет вам **индикатор грязного состояния** (кнопка с замочком / изменением):

- **Accept** — Сразу сохраняет внесенные изменения как новые константы в текущий пресет.
- **Revert** — Полностью отбрасывает (отменяет) все внесенные вручную правки в сцене и восстанавливает базовые значения сохраненные в пресете.
- **Accept & Sync** (++alt++-click по кнопке Accept) — Автоматически сохраняет и дополнительно синхронизирует (принудительно обновляет) каждое значение подвсех уровней, использующий именно эти же пресеты.

!!! note "Индикаторы Уровней (Tier Indicators)"
    Выпадающие списки пресетов имеют суффикс `[Tier]` на конце своего наименования (пример: `[Addon]` или `[User]`) для того чтобы вы могли с лёгкостью отличать встроенные пресеты аддона от файлов, созданных самими пользователями.
