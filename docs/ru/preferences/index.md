# Настройки (Preferences)

**Расположение настроек:** *Редактировать (Edit) > Настройки (Preferences) > Аддоны (Add-ons) > Takes for Blender*

Или вы также можете щелкнуть по **значку шестеренки (gear icon)** в самом верху заголовка Navigation Panel и нажать кнопку **Open Preferences**.

## Общие (General)

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| **Autosave Preferences** | Вкл (On) | Сохранять (переносить) все конфигурации пользователя в JSON данные, чтобы они сохранялись между сессиями/перезапусками Blender. |
| **Debug Logging** | Выкл (Off) | Активация логгирования по специальным категориям для записи в ежедневные физические файлы логов. |

## Функции (Features)

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| **Auto-Assign VL Actions** | Вкл (On) | Автоматически назначать "свободные" действия (каскадные действия анимации) индивидуально для каждого View Layer. |
| **Auto-Create Ref Action** | Вкл (On) | Автоматически зеркально (mirror) переносить и подставлять новые (созданные вручную кадры или измененные атрибуты) ключевые кадры прямиком в слой "Reference State". |
| **Auto-Rename Actions** | Вкл (On) | Автоматически и прозрачно изменять локальные названия анимаций для соответствия их общей каскадной спецификации наименований. |
| **Preset Automation Mode** | Вручную (Manual) | Каким образом будут обрабатываться все автоматические модификации с Пресетами при их изменении после установки (auto-accept, sync, manual). |
| **Batch Render Sound** | Отключено (None) | Системное аудио-уведомление (динамик, пиканье, звук) свидетельствующее о финальном завершении пакетного рендера. |

## Поведение (Behavior)

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| **Syntax Brackets** | Квадратные `[]` | Визуальный стиль символа ограничителя токенов, применяющегося для нужд Smart Output и именований файла. |
| **Separator Characters** | `sep1`=`_` | Настройка до 5 независимых знаков/символов "разделителей" (tokens). |

## Логгирование и отладка (Debug Logging)

Кода функция активирована, отладочная информация досконально пишется в посуточные лог-файлы с возможностями очень гибкой (покатегорийной) фильтрации (granular topic filtering):

| Категория (Topic) | Подкатегории (Subtopics) |
|-------|-----------|
| **Core (Ядро)** | VL Switch (Смена VL), Cascade (Движок каскада) |
| **UI (Интерфейс)** | Inspector (Поведение инспектора) |
| **Ops (Операции)** | Animation (Манипуляции с анимацией) |
| **Features (Функционал)** | Reference State (Отслеживание Базы), Variant Switch |
| **Render (Рендер)** | Batch Render (Очереди пакетного рендера) |

Плюс дополнительные тумблер-кнопки (Debug, Info, Warning, Error), способствующие фильтрации сообщений ещё до их строкового форматирования, экономя тем самым системные ресурсы программы.

!!! note "Местоположение лог файлов (Log Location)"
    Логи всегда сохраняются под маской имени `takes_for_blender_YYYY-MM-DD.log`
    в вашей настроенной директории. Все архивы (логи) старше 7 календарных дней система автоматически стирает (чтобы они не захламляли жесткий диск).
