## Wichtigste Funktionen

| Funktion | Beschreibung |
|---------|-------------|
| **Takes-Baum** | Einheitliche Hierarchie: Szenengruppen → Szenen → VL-Gruppen → Ansichtsebenen → Versionen. |
| **6-stufige Kaskade** | Überschreiben der Auflösung von der globalen bis zur Versionsebene für Kameras, Welten, Aktionen und mehr. |
| **Variantenwechsel** | Produktvariantensystem mit Materialwechsel pro Bereich, Einfügen von Voreinstellungen und Pool-Verwaltung. |
| **Rest State** | Automatisch spiegelndes Referenzaktionssystem, das makellose Standard-Keyframes über Animationen hinweg beibehält. |
| **Batch-Render** | Modale Render-Engine (Vorder- oder Hintergrund) mit einem Smart-Output-Token-System für die Dateibenennung. |
| **Render-Voreinstellungen** | JSON-basierte Voreinstellungen für Render-, Kamera-, Welt- und Dateiausgabeeinstellungen – pro Ansichtsebene. |
| **Inline-Vorschau** | Live-Viewport-Miniaturansichten neben jeder Ansichtsebene in der Baumstruktur. |

!!! tip "Blender-Version“
    Takes for Blender erfordert **Blender 5.1** oder neuer. Es nutzt das in Blender 5.0 eingeführte Slotted-Action-System.
