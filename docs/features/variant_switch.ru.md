# Переключатель Вариантов (Variant Switch)

Система **Variant Switch** управляет вариациями продукта — различными конфигурациями материалов, вариантами отделки и цветовыми решениями. Она отвечает за замену материалов в определенных областях и полностью осведомлена о состояниях каскада.

## Концепция

В визуализации продукции вам часто требуется отрендерить один и тот же продукт в различных отделках (Золото, Серебро, Матовый Черный цвет) или конфигурациях. Инструмент Variant Switch автоматизирует процесс подмены этих материалов.

## Иерархия

| Элемент | Описание |
|---------|-------------|
| **Продукт (Product)** | Главный контейнер верхнего уровня (например, "Часы", "Бутылка"). |
| **Состояние (State)** | Наименованный вариант продукта (например, "Золото", "Серебро"). Нажмите, чтобы включить предпросмотр. |
| **Деталь (Part)** | Компонент продукта, логически привязанный к коллекции (например, "Корпус", "Ремешок"). |
| **Пул (Pool)** | Слоты материалов, доступные для каждой конкретной Детали (part). |

## Использование

### Создание Продукта

1. Откройте панель **Variant Switch**.
2. Нажмите **+** или сочетание горячих клавиш ++ctrl+n++, чтобы добавить новый Продукт.
3. Деталь по умолчанию ("base") создастся автоматически с одним полностью пустым слотом пула материалов.

### Добавление Состояний

1. Выберите Продукт.
2. Добавьте новое Состояние (например, "Золото").
3. Каждое Состояние индивидуально хранит один индекс пула для каждой Детали, заставляя её в этом состоянии использовать нужный материал.

### Назначение Материалов

1. Разверните вкладку Детали (Part), чтобы увидеть её пул материалов.
2. Поместите (назначьте) в первый пустой слот нужный материал — сразу автоматически сгенерируется новый пустой слот.
3. Вы можете использовать для этого все стандартные функции Blender (New, Duplicate, Unlink).

### Предпросмотр Вариантов

Щелкните **значок ромбика (diamond icon)** над любым Неактивным Состоянием, чтобы оно немедленно применилось к модели прямо в окне Viewport. Обратите внимание, что текущее Активное состояние всегда отображается закрашенным кругом.

## Режимы Вариантов

Variant Switch официально поддерживает три рабочих режима подмены:

=== "Swap (Замена)"
    Прямое переназначение материала. Это самый простой режим — Материал А физически заменяется на Материал Б.

=== "Preset (Пресет)"
    Применяет сохраненный JSON пресет физических параметров поверх существующего материала. Сам материал на объекте остается нетронутым, но изменяются значения его нодов (свойства).

=== "Pool (Пул)"
    Осуществляет выбор из палитры перечня материалов на уровне продукта. У каждой детали (Part) имеется пул материалов, которые индивидуально индексируются у каждого Состояния (State).

## Интеграция с Каскадом (Cascade)

Состояния Variant Switch (Variant Switch states) полностью разрешаются вместе с прочей логикой каскада проектом. Это означает, что каждому View Layer (или на уровень выше) можно "жестко задать" или привязать свое активное состояние варианта, позволяя легко рендерить отличные друг от друга варианты продукта, не меняя ракурсы камеры, всё лишь за один клик.

!!! tip "Теги Вариантов (Variant Tags)"
    Вы можете назначать теги (из категории "Variant") для каждого состояния. Это полезно как для внутренней организации, так и для разрешения имён файлов системы Умного Вывода (Smart Output) через токен `{variant_tag}`.
