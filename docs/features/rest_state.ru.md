# Состояние Покоя (Rest State)

Встроенная система **Rest State** (состояние базового покоя, или "эталонное" состояние) автоматически сохраняет, страхует и резервирует все изначальные (чистые) значения ваших свойств параллельно с персональными анимациями на других View Layer слоях.

## Концепция

Когда вы анимируете один объект, но совершенно по-разному назначаете эти действия на каждом отдельном View Layer, для корректной подмены движку требуется «нейтральная», исходная базовая линия объекта — его базовая и изначальная точка положения, начальный угол поворота, родные значения материалов и т.п. Система "Состояние Покоя" (Rest State) создана поддерживать эту резервную базовую отметку без лишних усилий, всё происходит автоматически.

## Как это работает

1. Существует общий, единый для всего проекта блок действий под названием **Reference Action** (Эталонное Действие: `Reference_State`). В нём система тайно складирует все значения по умолчанию (нулевые отметки) абсолютно каждого анимируемого свойства строго на нулевом кадре (frame 0).
2. Делая любое действие по установке "ключа" анимации (keyframe) внутри любых View Layer слоёв, модуль подсистемы Rest State автоматически делает моментальную копию (зеркалирует/отражает) в этот блок Reference Action: туда закидывается текущее положение параметра объекта на этот момент, считая его за "базу".
3. Каждый раз выполняя переход между слоями View Layers, система проверяет: если на новом (целевом) слое у текущих объектов не обнаружена какая-либо назначенная анимация, объекты тотчас упруго «возвращаются» на свои базовые, немодифицированные позиции/модификации, хранящиеся в Rest State.

```mermaid
graph LR
    K[Keyframe Added (Добавлен Ключ)] --> M[Mirror to Reference (Отражение в Референс)]
    VL[VL Switch (Переход слоёв)] --> C{Has Cascade Action? (Есть ли каскадное действие?)}
    C -->|Yes (Да)| A[Apply Action (Применить Анимацию)]
    C -->|No (Нет)| R[Snap to Rest State (Вернуть в Состояние Покоя)]
```

## Элементы управления

| Элемент управления | Расположение | Описание |
|---------|----------|-------------|
| **Auto-Mirror (Авто-зеркалирование)** | Заголовок панели / Глобальные параметры | Включение и остановка автоматического базового зеркалирования во время установки вами ключевых кадров анимации. |
| **Set Reference Default (Установить Базу Покоя)** | Контекстное меню мыши / ++shift+alt+i++ | Принудительная перезапись и ручная отправка текущих активных значений системы как новое "базовое" значение для Rest State. |
| **Revert to Rest (Вернуться к Покою)** | Горячие клавиши ++alt+i++ | Привязка (сброс) выбранного параметра свойства строго обратно для перерасчета к его безопасному значению в Rest State. |

## Поддерживаемые Блоки Данных (Supported Datablocks)

Система Rest State способна контролировать, поддерживать и восстанавливать все стандартные анимируемые блоки данных программы:

- Физические объекты (ориентация осей, видимость)
- Типы света (их мощность, окрас, размер)
- Объекты камер (фокусы и расстояния, глубина DOF)
- Свойства материалов (задания шейдерным нодам)
- Мировое окружение (атмосферные параметры world)
- Свойства сцен (гравитационные силы, длина таймлайна)
- Нодовые связи Деревьев (шейдеры материалов, связи композитора)

!!! warning "Формы Ключей и Кости объектов (Shape Keys & Pose Bones)"
    Анимационные значения ключей форм (Shape key) для мешей и внутренние повороты костей позирования (pose bone transforms) в данный актуальный момент пока еще технически не поддерживаются системой восстановления параметров (Rest State system). Данный функционал стоит в графике добавления уже в следующих грядущих версиях аддона.
