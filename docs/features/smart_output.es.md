# Smart Output

El sistema de **Smart Output** (Salida Inteligente) proporciona una resolución dinámica de rutas de red y formatos renderizados según cada token interno específico. Modificando de esta forma el nombre que tendrías en modo singular aplicando el renderizado del producto en tiempo y modo constante, a resoluciones únicas de carpeta.

## Uso a un Clic en Opciones Principales

1. Dirígete lateralmente y da tu permiso de activación del recuadro principal dentro de las pestañas genéricas globales (opción **Output**).
2. Identifica tu switch interactivo de nombre: **Smart Output**.
3. Personaliza tu **Directory Pattern** (Regla Principal Intermedia del Patrón de Ruta) mediante el sistema token.

## Sintaxis de Uso Abierta de Código 

Esos pequeños "tokens" que representan datos deben ir recubiertos. De manera ordinaria a través de la instalación por default verás su uso encapsulado en modo cuadriculado por tu configuración de Blender original:

```
[view_layer]_[camera]_####.[file_format]
```

Todo eso significa que lo expuesto finalmente sería derivado a un nombre predeterminado y físico final como: `Front_3-4_CamHero_0001.png`

### Reglas Básicas Personales

Decide tú mismo su modo nativo y directo modificando un switch nativo individual. Busca la configuración integrada principal bajo el marco final que es **Addon Preferences > Behavior Options > Syntax Brackets**. Opcionalmente cambiará su diseño bajo 6 valores de preselección visual general nativa del addon en la aplicación original principal a ti mismo: 

| Tipo | Esquema Base | Regla de Nomenclatura Estricta General |
|-------|--------|---------|
| Corchete/Square | `[token]` | `[view_layer]_####` |
| Curvado | `{token}` | `{view_layer}_####` |
| Angular/Flechas | `<token>` | `<view_layer>_####` |
| Circular | `(token)` | `(view_layer)_####` |
| Signos base / % | `%token%` | `%view_layer%_####` |
| Divisas | `$token$` | `$view_layer$_####` |
| Almohadilla | `#token#` | `#view_layer#_####` |

## Separaciones Físicas y de Estructuras 

Integra como quieras modos variables únicos limitados preestablecidos. Al sumar 5 códigos independientes de este sistema base (`[sep1]` integrando la cadena individual base consecutiva originaria en final hasta llegar la opción conectada local originaria: `[sep5]`):

- `[sep1]` = `_` Para diferenciar el apartado por tu bloque al generar una sección inicial base originaria en el modo final del formato primario y del resto general para la creación completa base de directorio principal y demás bases originales de archivo base/archivos creados bajo su formato físico principal originario/carpeta originaria base. 
- `[sep2]` = `-` Modo con guión inicial para variables separadoras interactivas inicial base secundaria/variables secundarias separadoras bases interactivas finales conectadas.
- `[sep3]` = `.` Usado normalmente inicial base de origen principal nativo conectando la numeración inicial conectando versiones originarias de base final.

Adicionalmente, se permite editar predeterminadamente y localmente (configura el archivo separador local) cada signo nativo final y nativamente editando de regreso su modo sobre tu espacio principal de **Addon Preferences**.

## Uso Oficial Interconectado Físicamente General Inicial Con Fotogramas/Padding de la Secuencia (Frame Numbers Inicial Globalizado Original)

Blender mantiene tu secuencia pre-aprobada bajo este predeterminado símbolo cuadruple especial y con separador especial de origen de ceros `####` permitiendo su escritura base sin problemas mayores de manera original al origen y sumada base de numeración pre-ajustada oficial para su estructura y edición; pero el addon también permite interconectarla con una visión oficial y previa en vivo de manera interactiva visualizándolo activamente durante tu lectura visual original.
