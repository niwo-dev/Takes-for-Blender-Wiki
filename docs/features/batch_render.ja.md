# バッチレンダリング (Batch Render)

**バッチレンダリング (Batch Render)** システムは、各 View Layer に個別に設定されたカスケードからの広範なオーバーライド群（各専用カメラ、各専用ワールド環境、各アクション、各種プリセット、バリアント指定など）をすべて反映させた上で、複数の View Layer をまたいだ包括的かつ連続的なレンダリング作業を完全に自動化・代行します。

## 対象のレンダーモード (Render Modes)

Takes for Blender は、2つの異なるレンダリングモードを完璧にサポートしています：

=== "フォアグラウンド (Foreground)"
    現在起動し、目の前で開いているその Blender のセッション内部で直接レンダリング処理を走らせます。レンダーウィンドウと見慣れた進行状況がリアルタイムで目の前に表示されますが、代わりにレンダリング実行中の間は Blender のその他の操作はロックされます。

    - ツリーのサイドバー下部にある **Render (レンダリング)** ボタン (:material-image:) をクリックします。
    - 進行状況は VL ごとに状況ステータスインジケーターとして詳細に表示されます。
    - キーボードの ++esc++ キーを押すと、任意のタイミングで一括キャンセルできます。

=== "バックグラウンド (Background)"
    完全に独立した「ヘッドレス (GUIを持たない) Blender サブプロセス」らを水面下で生成して、そこでレンダリングを実行させます。これにより、バックグラウンドでのレンダリングが淡々と進行している最中であっても、目の前の Blender 本体はフルスピードでアクティブに保たれ、完全に自由に動作・作業し続けることが可能です。

    - サイドバー下部にある **Desktop (デスクトップ)** ボタン (:material-desktop-classic:) をクリックして実行指令を出します。
    - 各VLの進行に従い、ツリービュー上の進行状況が自動的に順を追って更新されてゆきます。
    - キューに積まれた全てのタスクが終了すると、作業の完了を告げる通知音が鳴り響きます。

## レンダー待機キュー (Render Queue)

レンダリング実行中のキューでは、各 View Layer の現在の作業ステータス状況が表示されます：

| ステータス状況 | 説明 |
|--------|-------------|
| **Pending (待機保留中)** | 順番が回ってくるのを待機している状態。 |
| **Rendering (レンダリング中)** | まさに現在・絶賛処理中。 |
| **Saving (出力保存中)** | 出力された結果ファイルをディスクに書き込んでいる作業中。 |
| **Done (処理完了)** | 何のトラブルもなく大成功で完了。 |
| **Failed (痛恨の失敗・エラー)** | 何らかのエラーが発生して頓挫 (ホバーすると詳細なエラー原因を見られます)。 |
| **Cancelled (強制キャンセル)** | バッチ全体が人為的にキャンセルされて中断されたため、スキップされた状態。 |

## 各選択モードと対象の指定 (Selection Modes)

- **Single View Layer (単一のVL出力)** — 大前提のデフォルトの動作。現在アクティブで選択中のただ 1 つの View Layer のみを対象にレンダリングします。
- **Multi-select (複数・一括選択モード)** — パネルの統計列にある専用トグルを有効化させ、複数選択モード (マルチセレクトモード) に入っている場合は、チェック（選択）されているすべての View Layer 達を順次レンダリング列へと放り込みます。

!!! tip "レンダリングの進行順序 (Render Order)"
    バッチレンダラーは「（あなたが画面上で目視で見ている）ツリービューの表示順（等しく上から下への順・Top-to-Bottom）」にきっちり従って処理を進めます。
    Blender内部データ上の難解なシーン/VL順序がどうなっていようが、一切無視され影響されません。

## クラッシュ時等の復旧手段 (Recovery)

万が一、原因不明のエラー等によりバッチレンダリングが「スタック (身動きが取れない状態)」に陥ってしまった場合：

1. **++alt++ + クリック** で、Render (レンダリング) ボタンを押し込み「強制リセット」を実行してください。
2. これにより、システム内部に残ったままになっている全ての待機フラグ指定が完全除去され、抑圧を受けていた各種イベントハンドラ機能が即時・安全に救済・復元されます。

## 出力機能との強力連携 (Output)

各種の保存先・出力パスは、[Smart Output (スマート出力)](smart_output.md) トークンシステムを介して見事に解決されます。各View Layer が算出したレンダリング結果出力は、あなたが設計した強固な出力パターンのルールに基づき「完全な自動処理」で名付けられて出力されていきます。
