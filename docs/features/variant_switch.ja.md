# バリアントスイッチ (Variant Switch)

**バリアントスイッチ** システムは、影響を及ぼすスコープ（効果範囲）ごとのマテリアル置換機能を備えており、カスケードシステムと手を取り合ったステート管理によって、あなたの製品の様々なバリアント（多様な素材構成、表面仕上げの違い、あるいはカラーオプション等）を強力に統括管理します。

## コンセプト

プロユースの製品ビジュアライゼーションにおいては、同一の製品を複数の「仕様」や「仕上げ」（例：ゴールド、シルバー、マットブラック等）、あるいは異なるパーツ構成で連続レンダリングする必要に頻繁に迫られます。Variant Switch は、そうした煩雑なマテリアルの交換（スワッピング）処理プロセスを完全に自動化します。

## 階層構造 (Hierarchy)

| エレメント | 説明 |
|---------|-------------|
| **Product (製品本体)** | バリアントツリー最上位のコンテナ大枠 (例：「腕時計全モデル」, 「ボトル本体」)。 |
| **State (状態・仕様)** | 製品の個別の派生仕様名 (例：「ゴールド版」, 「シルバー版」)。アイコンクリックでプレビュー可。 |
| **Part (各連携部品)** | コレクション (Collection) に深くリンク・紐付けされた個別のコンポーネント部品 (例：「ボディー筒」, 「ストラップ帯」)。 |
| **Pool (ストック在庫)** | それぞれの Part において任意に選べる、複数のマテリアルスロットの在庫群。 |

## 使用方法 (Usage)

### Product (プロダクト) の作成

1. **Variant Switch** パネルを開きます。
2. **+** をクリックするか ++ctrl+n++ を押して、新しい Product (製品本体大枠) を追加します。
3. デフォルトの Part (部品名："base") が、1つの空きプールスロットをもった状態で自動的に作成されます。

### States (仕様ステート) の追加

1. その Product を選択します。
2. 新しい State (ステート: 例「ゴールド仕様」等) を追加します。
3. 各々の State は、「Part ごとに用意されたプールの中から『何番目のインデックスのマテリアル』を採用して使用するか」を自身の中に記憶します。

### マテリアルの割り当て (Assigning)

1. 配置された Part を展開拡張すると、そこに紐付くマテリアルプール (Material Pool) が一気に見えます。
2. その最初の「空欄 (Empty)」のスロットに、適当なマテリアルを割り当ててみてください ── すると直ちに、次の新しい空きスロットが一つ自動生成されます。
3. そこからは、Blenderに標準搭載されているネイティブ機能側のマテリアルセレクターの作法 (新規作成、複製、リンク解除 等) を普通に使用して作業できます。

### バリアントの即時プレビュー (Previewing Variants)

非アクティブな状態である State の横に配置された **ひし形アイコン (diamond icon)** をクリックすれば、その仕様のバリアント構成が直ちにビューポート上に適用・プレビューされます。なお、現在アクティブになっている（適用中の権限を持つ）ステートは、中身が塗りつぶされた完全な円形で表示されます。

## バリアントのモード挙動 (Variant Modes)

Variant Switch は、以下の 3つの挙動モードをサポートしています：

=== "Swap (スワップ)"
    直接的なマテリアルの置き換え。「マテリアル A が、マテリアル B になる」という最もシンプルで分かりやすいモード。

=== "Preset (プリセット)"
     JSON を介して行われる、設定値インプレイス（その場）でのマテリアルプリセットの注入。「適用されているマテリアル自体は全く同じもののままだが、その中のプロパティ値や色設定が上書き変化する」という特殊なモード。

=== "Pool (プール)"
    プロダクト・レベルに設けられた巨大なマテリアルパレット (絵の具皿) から、指定番手を取り上げて使っていくモード。各 Part （部品）が専用のマテリアルプール在庫を持ち、ステートの切り替えによってそのインデックス番号が呼び出されます。

## カスケードシステムとの統合 (Integration with Cascade)

Variant Switch 内の各 State（状態名）は、巨大なカスケード群の一部としても見事に解決されます。各 View Layer（またはそれより上の高い階層）は、「どのバリアントの State をアクティブ化して呼び出せ」と個別に指定できるため、カメラアングル（カット）ごとに、自動的に全く異なるバリアント構成を連鎖的に引き起こさせることが可能となります。

!!! tip "バリアント用タグの設定 (Variant Tags)"
    カテゴリ内の「Variant」タグを固有のステートに割り当てることで、プロジェクトの整理整頓だけでなく、
    `{variant_tag}` トークンによる自動解決を用いた、強力な Smart Output 自動名付け出力をコントロールできます。
