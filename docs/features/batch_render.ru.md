# Пакетный рендер (Batch Render)

Система **Batch Render** автоматизирует рендеринг через множество View Layers, методично применяя ко всем каскадные переопределения (камеры, окружения, анимации, пресеты и варианты) перед их рендерингом для создания конечного результата.

## Режимы Рендеринга

Takes for Blender на текущий момент поддерживает два режима:

=== "Foreground (На переднем плане)"
    Просчёт проекта происходит внутри текущей активной сессии Blender. Вы видите окно финального рендера и следите за прогрессом в реальном времени, однако во время этого процесса интерфейс Blender блокируется (замораживается) системой.

    - Нажмите кнопку **Render** (:material-image:) на боковой панели дерева.
    - Прогресс показывается для каждого индивидуального VL через индикативные статусы.
    - Нажмите ++esc++, чтобы отменить всё.

=== "Background (В фоновом режиме)"
    Система выполняет все рендеры как отдельные "headless" (невидимые) процессы Blender (subprocess). Текущий Blender остаётся полностью интерактивным под вашем контролем, пока рендеры отрабатывают в фоновом режиме вашего ПК.

    - Нажмите кнопку **Desktop** (:material-desktop-classic:) в боковой панели.
    - При завершении каждого VL графическое древо в панели будет постепенно обновляться миниатюрами.
    - При завершении абсолютно всех задач воспроизведется звуковой сигнал завершения.

## Очередь Рендера

Очередь рендера демонстрирует состояние для каждого отдельно взятого View Layer:

| Статус | Описание |
|--------|-------------|
| **Pending (В ожидании)** | Ждёт своей очереди для начала отправки на рендеринг. |
| **Rendering (Рендеринг)** | В настоящее время активно визуализируется процессором/видеокартой. |
| **Saving (Сохранение)** | Сохраняет готовый итоговый выходной файл на жесткий диск. |
| **Done (Готово)** | Операция успешно закончена. |
| **Failed (Ошибка!)** | Произошла ошибка (наведите курсор, чтобы прочитать подробности). |
| **Cancelled (Отменено)** | Пропущен из-за отмены пакета пользователем. |

## Режимы Выбора

- **Один View Layer** — Рендерит только активный View Layer (по умолчанию).
- **Множественный выбор (Multi-select)** — Включая параметр "мульти-выделения" (multi-select), система рендерит только те View Layers, на которых стоят галочки выделения.

!!! tip "Порядок рендеринга (Render Order)"
    Пакетный рендеринг выполняет очередь сугубо в том порядке, который отображается в самом дереве панели Takes (сверху-вниз), и игнорирует внутренний порядок Blender для сцен/слоев с их списками.

## Восстановление (Recovery)

Если пакетный рендер завис или "застрял":

1. Сделайте команду **Alt+Click** (Клик мышью + Alt) по кнопке Render, чтобы принудительно перезапустить систему.
2. Это очистит все внутренние флаги интерфейса и восстановит все скрытые обработчики системы.

## Вывод файлов (Output)

Выходные форматы и пути формируются системой токенов [Smart Output](smart_output.md) (Умный вывод). Каждый вывод слоя View Layer получает свое уникальное автоматизированное имя на основании выбранных тональностей настроенного паттерна.
