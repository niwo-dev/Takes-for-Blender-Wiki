# 渲染参数预设档 (Render Presets)

**Render Presets(渲染参数预设档)** 说穿了本质就是对整个 Blender 关于出图环境设置属性所保存生成一份类似文本记忆档案的超级快照系统 (采用跨平台最优的 JSON 格式生成存储)! 这些个被提取保存好之快照能够再次依赖本插件系统的超级传输带：我们的大系统体系 "Cascade"，将所有该设置如同血液分发到各个所辖末端各色各样的 View Layer。

## 快照分类品种大览 (Preset Types)

| 这是关于什么的设置包 | 它偷录和复印下来哪些记忆范围 |
|------|-----------------|
| **Render (总渲染大厅)** | 各式引擎种类、抽样渲染精度 (samples)、整体产出长宽比例、颜色后期调和管理, 和关于底片的设置集合。 |
| **Camera (摄像系统)** | 镜组长、传感器体量和尺寸标度、光学虚化能力、还有什么前后可收涉裁切视野边界等等... |
| **World (大气天下环境)** | 空中打灯和氛围渲染度, 各种布景反光的色系统统等参数全景收集、包括高深AO设定。 |
| **File Output (导出产出设置包)** | 图片最后是什么类型的图像(jpg, png或多通道exr!) 精度、深度甚至带透明底不带、以及如何对位投放到Smart Output设定好的那个路线。 |

## 从何创立自造预置 (Creating Presets)

1. 请随便按你素日做事的偏向好在右边系统自己原始的地方去更改配置妥当你一切想要留下来保存备份的值等。
2. 切回我们体系的界面并翻点对应于树里的每一行末尾属于它的"工具箱选项"去打开浮层。
3. 把里边的 **Save as Preset (储存此参数做预选单)** 点下后给这份参数想个方便好理解的中文名！
4. 那这就好啦！你辛辛苦苦调通出来的数值组现已成了躺平在你家软件库 (user presets directory) 里的标准可读档案 (JSON文件)。

## 下达差遣部署预置档 (Assigning Presets)

由于咱们有万能派发利器 "Cascade(大瀑布级联)": 这预设配置你把它随便安置发往任意层次随传随到:

- **Global（神域之全局）** — 控制整套包场所有活计和全部底下干事的终极初始最高阶设置模板集！
- **Scene Group (大区群组)** — 区域网老大，仅对归并到麾下该系列的所有跟拍情景实施强制全通用这套模版。
- **View Layer (特定特调场)** — 这个镜头单独需要最高级精度规格。
- **VL Version (枝微末节之修订本)** — 这个小快照就稍微在刚刚镜头的基础上微微动点歪心思！

## 本机脏污涂改警示判定 (Dirty State)

当你好死不死已经调用套入了一大套完完整整理想快照参数的时候，手欠又或者被形势所逼被迫在外围稍微手残对一两处渲染设定微修。这时候界面即刻升起一个 **发黑点(dirty indicator/红点提示)** 警告并盘问你: 

- **Accept (承认且保存)** — 把这笔偷偷更改覆盖吃掉入原来的正餐快照档并就此保留此行径作为该份档案的全新正确姿势!
- **Revert (清算且滚回之前)** — 这叫破镜重圆! 直接推翻之前做的一切杂乱手工修改回滚复原成最核心一开始的参数模板档。
- **Accept & Sync (强行收下且逼人接受)** (需叠加 ++alt++-同下结合) — 完全修改且保存到原本该设定内去外加还要通发四方，把凡事关联跟过该预设名头的全都给大变天更新。

!!! note "身份档标签尾缀阶层辨明 (Tier Indicators)"
    你可以一目了然凭借各种设定旁边缀有的后缀譬如显示为 `[Addon]` 还是说叫做 `[User]` ，用来无痛划分辨识出到底这选项是开发商良心免费内植的默认设定组合，亦或是身为超级主人的你所亲手写下的定制套组件库档案!
