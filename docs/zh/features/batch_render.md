# 批量渲染 (Batch Render)

**Batch Render (批量渲染引擎)** 则毫无悬念负责起包揽且自动化帮你一口气渲完一切多个处于不确定性下 View Layers 数据流的所有活。并且在此过程里稳若泰山地针对不同层面派送其理应当获取的各类参数覆层：摄影机位怎么定、世界咋发光、到底这层在作何种动作运动和各类变异材料统共不在话下。

## 两大工作派系模式 (Render Modes)

Takes for Blender 在发令渲染时只吃如下2家派系模式：

=== "Foreground (前台阻断式硬刚)"
    这就仿佛你在 Blender 内点了一次出图。你只能傻傻盯着这套大屏幕一直跳动进程条、眼看显卡转干出成片，你的任何在程序的试图越狱造反干预都会被无视并且剥夺所有一切控制权 (locked)。

    - 在侧边工具条直接压下那个 **图片板按钮 Render** (:material-image:) 去激发。
    - 中控台条列将为每一道分别的 VL 放各自独立的读条来供你检验进度的快感。
    - 半道遇不爽可以随时一巴掌敲下 ++esc++ 终止这一连片作业序列。

=== "Background (极其深幽的幕后操作法)"
    该手法会在完全没有任何提示之下，悄悄潜入调动生成多个并行的 “无头版(Headless)” Blender 小鬼核心为你底层打黑工! 所以，你的主 Blender 程序在这边活蹦乱跳并且响应你任何极端的乱修妄动操作都不会因此瘫痪干扰卡顿!

    - 点击那个像个 **复古老式电视机（Desktop）** (:material-desktop-classic:)的图标。
    - 你唯一能洞晓到进度的方法就是在主面板列表边小图标上不停看见各处进度跳完变成成果照片墙!
    - 当然贴心附赠: 当系统彻底把堆积的渲染债解决掉瞬间, 它必定会来一记声控大叫通知（系统完成提示音）吵你一下告诉该下班收图啦。

## 渲染队伍队列监控 (Render Queue)

队列列表随时反馈每一列层 (View Layer) 当打的状态境遇:

| 状态文字 | 表示及解释 |
|--------|-------------|
| **Pending (队列之中等待临幸)** | "没开始排队等着呢" |
| **Rendering (玩命生成出图中)** | 引擎正冒热气疯狂产出中! |
| **Saving (落袋为安存盘中)** | "马上好, 正拿硬盘狂抄存入照片文件里！" |
| **Done (安全落幕完成!)** | 过审完毕且任务圆满结束。 |
| **Failed (血崩阵亡!)** | 有东西不对劲,出了差错(把你的鼠标挪到上面会抛给你阵亡诊断原因详解)。 |
| **Cancelled (强行被砍!)** | 要么由于直接把总闸拉断中止导致这层错失渲染，或者主家打脸给活生生切断罢工之原因。|

## 选择何者上台参演 (Selection Modes)

- **单兵单打 (Single View Layer)** — 这是基础模式，只出你眼皮子底唯一高亮处于正活动中那位 View Layer而已 (缺省自带模式)。
- **大军出动混战法 (Multi-select)** — 如果侧边那个长得像 "空心盒子带复选的方格项（multi-select）" 被彻底打开后。大权便移送于所有带了钩打上对钩印记者全体！并把它们做串烧全部推下去一网打尽处理。

!!! tip "出将顺序榜法门 (Render Order)"
    本插件做分派是绝对根据你所**看得见的系统列表上从顶楼倒向底楼（从上而下）**的视觉排序先后队列来进行计算推进的。
    **绝非**照什么背后隐藏核心数据的内部鬼怪序列表。

## 中山脱险与拔线救援 (Recovery)

一旦你的出图队列惨遭离奇"暴死卡住发癫疯"的奇案：

1. **组合键死力强压 Alt+左击点按 (Alt+Click)** 那个最大的渲染启动主旋钮来使之彻底崩坏断电清空重生(force-reset)。
2. 这暴力之法会无理强横洗尽所有处于底端之标志与那些死活没解除干净死咬处理的乱序挂链钩子。

## 收成果实 (Output)

成图导出放置均靠一套庞杂精妙名唤 [Smart Output (灵性输出口)](smart_output.md) 之自动变量符号规则来进行终极导航及命名的工作任务。每有一位 View Layer 闯关结束它的成品就必能得一匹配其身定做生成的美艳不撞款的档案大名。
