# 变体树 (Variant Tree)

**位置:** *Properties Editor (属性编辑器) > Takes 选项卡 > Variant Switch*

Variant Tree（变体树）专用于管理所有的产品变体 —— 不同的材质配置项、不同的色系或者完全迥异的产品状态。系统会非常硬核地以 Product (大产品)、State (细分状态) 和 Part (部件节点) 作为构建方式。

## 层级结构

```mermaid
graph TD
    P[Product (大产品)] --> S1[State A (配色 A)]
    P --> S2[State B (配色 B)]
    S1 --> PA[Part: Body (部件: 瓶身)]
    S1 --> PB[Part: Cap (部件: 瓶盖)]
    S2 --> PC[Part: Body (部件: 瓶身)]
    S2 --> PD[Part: Cap (部件: 瓶盖)]
    PA --> M1[Material Pool (备用材质池)]
    PB --> M2[Material Pool (备用材质池)]
```

产品 (Product)
:   最大的顶层收纳容器 (例如：某款 "手表", "保温杯" 项目整体)。

状态/款式 (State)
:   当前这款产品所具备的实名变种外观款式 (比如：主打的 "红金", "银灰", 或者 "哑黑")。

部件 (Part)
:   一件产品的组件或组件结合，通过 Blender 自有的集合绑定形式来呈现作用区域 (涵盖 "表盘", "表带" 之类的物件)。
    且所有的部件自出生就必须带上独立的索引位空位库作为池子以供选料配菜。

## 基本使用方法

### 切换产品变体

将鼠标轻松点击未开启状态文字前面的这枚小小的 **菱形钻石形图标 (diamond)**，系统立刻将在 3D 界面内帮你实时演算与更换对应皮肤供你观赏评估。当前正在展现给你的皮肤总是用填实的**圆圈标志**来告诉大家它在场。

### 材质池系统 (Material Pool)

每一个 Part (部件) 自带一套独立隔离的"换料菜单"（材质选项池）—— 就是一份能随手抓材料替掉默认素材的快捷清单:

1. 先在首个系统创建的初号空洞插槽里推入任何你喜爱的材质（随即下方会在背后悄无声息诞生下个备用口子等候差遣）。
2. 调用它前面挂着的 **编号数字 (pool index)** 号码牌去命令指派目前属于部件上的特定主位选项。
3. 当你来回按键变更 "状态款式 (State)" 的过程中，我们的引擎在底下已经翻箱倒柜把对应编码的匹配参数和材质准确扔出匹配完成替换工作了。

### 集合的指派应用 (Collection Assignment)

必须得让每个登记的 Part 与真实世界的 Blender collection（物料模型对象集合包）建立死扣。但凡扔进来这个口袋的随便一件 3D 物体或者是它的子集附带元素，在此刻统统均会接收到该覆盖并直接"换下对应衣服色系"。

## 变体专属签子 (Variant Tags)

别忘记对各种样式 State 加入分类系统里的特有签 ("Variant" 型签名分类)，在做规整项目档案的基础上还能享受来自后期 Smart Output 的全自动化导出支持。借助系统给的特殊变量符 `{variant_tag}` 即可是实现自动化批量分类！
