    Changelog¶

    All notable changes to this project will be documented in this file.

    The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

    Legend: 🟢 confirmed working in-app  ·  🔴 added but not yet verified in-app

    [Unreleased]¶

    Added¶

    1. 🟢 The add-on's labels and tooltips can follow Blender's language setting — German is the first pilot language, starting with a handful of labels while the full translation is filled in.
    2. 🟢 Internal: a developer tool gathers every piece of add-on text for translation — no behavior change.
    3. 🟢 The add-on's side panel now shows up in German — panel titles, tab and section headers, list menus and keyframe pies.
    4. 🟢 Internal: an automated check catches wording changes that leave a translation behind, and flags new text that has no translation decision yet — no behavior change.
    5. 🟢 Nearly the whole interface now speaks German — preferences, buttons, menus, warnings and short tooltips; only the longer explanation texts are still English.
    6. 🟢 The Autokey button now handles Blender 5.2's "Only Insert Available" setting for you — it asks once for permission, turns the setting off while Autokey is on so fresh takes key immediately, restores it after, and backs off with a warning instead if you manage the setting yourself.
    7. 🟢 The German translation is complete — every label, tooltip and message the add-on shows now has German wording, so switching Blender to Deutsch no longer drops you back into English mid-panel.
    8. 🟢 Messages the add-on builds on the fly now translate too — every dialog heading, warning header, confirmation and status line composes its German around the inserted name or count, including correct singular and plural wording ("1 Kaskaden-Abweichung" / "3 Kaskaden-Abweichungen").

    Changed¶

    1. 🟢 Internal: removed a leftover unused helper from the Rest State cache — no behavior change.
    2. 🟢 Internal: a test that assumed English datablock names now passes on non-English Blender installations — no behavior change.
    3. 🟢 Internal: the add-on's info-box texts can now carry manual line breaks — no visible change until a text uses one.
    4. 🟢 Internal: 18 tooltip and info-box texts moved into the central texts folder, so their wording lives in one editable place — no visible change.
    5. 🟢 Internal: a further 143 property tooltips moved into the central texts folder — no visible change.
    6. 🟢 Internal: text that lives in the central texts folder is now picked up for translation; roughly 500 already-centralized strings had been invisible to the translation collector.
    7. 🟢 Internal: every operator tooltip now lives in the central texts folder instead of being taken from a developer comment in the code — same wording on screen, but the text is editable in one place and translatable for the first time.
    8. 🟢 Menu tooltips (the dropdowns' hover text) are now centralized and translated too — they previously showed only in English because they lived in the code rather than the texts folder.
    9. 🟢 The last multi-line property tooltips are now centralized in the texts folder too — no visible change (they were already translated), just editable in one place. Only a handful of tooltips built at runtime remain in code.
    10. 🟢 The very last tooltips built at runtime are now centralized too, so every piece of UI text lives in the texts folder — and the two storage-warning boxes translate as well.
    11. 🟢 Status and error messages the add-on shows after an action can now be translated too (Blender never translates those automatically) — all 651 are routed through the translation system; the German wording is filled in next.
    12. 🟢 Status and error messages now appear in German too — all 525 are translated, so a German user sees German feedback after every action, not just German buttons and tooltips.
    13. 🟢 Tooltips, setting descriptions and info boxes read cleaner across the whole add-on — stray line breaks and double spaces are gone, punctuation and terminology are consistent (View Layer, Alt+Click), developer notes no longer leak into hover text, and the German translations follow the new wording.

    Fixed¶

    1. 🟢 Clearing an object's keyframes no longer traps it: moving it afterwards stays put instead of snapping back to rest after every move, objects sharing the same take are left alone, and the snap that fires on a genuine keyframe deletion lands on the actual rest pose instead of a stale spot.
    2. 🟢 The "Auto-snap on Keyframe Clear" preference now actually turns the automatic snap off — it previously did nothing.
    3. 🟢 The New Slot button in a datablock's action popover creates the slot — it previously always failed with a Python error.
    4. 🟢 Right-click → Online Manual on the two autokey-setup buttons now opens the section that documents them, instead of dropping you on the wiki home page.
    5. 🟢 The two autokey-setup buttons grey out when the add-on's preferences can't be read, instead of staying clickable and quietly doing nothing.
    6. 🟢 Buttons in the add-on now translate too, not just panel headings — the Context and Inspector tabs read Kontext and Inspektor in German.
    7. 🟢 The custom-token preset picker reads like the other preset rows — a searchable dropdown, plus create, rename and delete buttons once a preset is chosen.
    8. 🟢 Info boxes now show in your language — the explanatory text inside tip and warning boxes was staying English in a translated interface because it was word-wrapped before translation; it's now translated first.
    9. 🟢 Starting a batch render on Blender 5.2 no longer crashes Blender instantly — a Blender 5.2 bug hands add-ons an empty render-statistics message the moment a render starts, taking the whole application down before the add-on can react; the add-on now stays off that channel on 5.2 and the progress bars use the time-based estimate instead.

    18. 🟢 The Context panel warns when a naming template has a wrong-format or invalid token, with one-click fixes; a broken template falls back to its default so names keep generating.
    19. 🟢 The render queue remembers each take's recent render times together with the settings they were measured at, and rescales its estimates when resolution or samples change.
    20. 🟢 A Value Lock button in the navigation header protects the scene's unkeyed values — objects, materials, lights, the world, and everything else that can hold a static value. While it's on, nudging an un-animated value on your selected objects snaps it back to what it was when you locked — guarding against accidental global changes that would apply at every frame. Keyframed (animated) values stay fully unlocked, and deliberately keying a value takes it out of the lock. Autokey pauses while the lock is on (its button dims) and returns on unlock. Take switching keeps working while locked, on any view layer, with or without an action.
    21. 🟢 A Rest State Mode button in the navigation header switches every view layer to show the rest baseline — compare it against your work or adjust it in place, then click again to return; tree assignments and renders always keep the real actions, and autokey pauses while it's on.
    22. 🟢 A status banner near the top of the 3D viewport shows a colored pill for each active mode you choose to surface — Autokey, Value Lock, Rest Mode, Still Mode, Timeline Sync — plus a progress bar for each running background task (view-layer preload, batch render), so a mode or a busy pause is visible at a glance when the Takes panel is closed; pick which mode pills appear, place the banner at any corner or edge with an X/Y pixel nudge and an opacity slider to fine-tune, hide the modes or the progress rows independently, or turn it off under Preferences → Interface → Overlay.
    23. 🟢 A Still & Animation mode, set globally from the navigation header or per take. Still pins a take's timeline to a single frame — start and end both at 0 — so you can build and light a still without scrubbing off or dropping a keyframe on the wrong frame; Animate gives a take its own start/end range; Default inherits from the level above. Click a take's mode icon for a quick Still / Animate / Clear dropdown, or Shift-click for a Timeline Options popover that sets the range and a pinnable default frame the playhead jumps to when you switch to that take — a viewport convenience that never changes what renders and is overridden while Timeline Sync is on. Renders of a Still take render frame 0, autokey stays on since any key just lands on frame 0, changing one take's mode never moves another take's timeline, and it works alongside Value Lock and Rest State Mode.
    24. 🟢 A Pacing setting (Gentle / Balanced / Fast) chooses how hard the view-layer preloader rests between layer builds.
    25. 🟢 A new Icon Sheet browses every Blender icon by category or search, copies an icon's name to the clipboard when you click it, and keeps a Recent tab of the icons you've used most recently; open it from a hotkey you assign — or a button — under Preferences → Developer → Utilities.
    26. 🟢 The switch profiler shows a Camera Visibility row — whether the camera automation fired on that switch, and how long it took.

    Added¶

    1. Slot bindings now survive renames across undo, file load, and action duplication. The VL switch records a stable token on every target ↔ slot pair it binds, so a future switch can rediscover the binding by identity even if the target was renamed in between — including renames that happened across an Undo / Redo / file open / Blender restart, which the existing rename detector misses because its pointer-identity cache gets flushed on those events.
