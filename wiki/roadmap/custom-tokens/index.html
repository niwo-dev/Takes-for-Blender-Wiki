    Custom Tokens¶

    Shipped

    Done and in the add-on.

    Theme: Feature · Area: Core · Size: 🔴

    Let users define their own Smart Syntax tokens, not just the built-in ones — give a token a name, point it at the scene data it should read, and then use it in output paths exactly like a native token. You build a token by picking a data source (scene, view layer, render, camera, or the current date/time) and the value to read from it, with an optional format — there's no code to write. Each custom token is checked the moment you define it (valid or not, with a live preview of what it currently resolves to), and resolution runs in a safe, sandboxed way so a bad or hand-crafted definition can never crash Blender or run unrestricted code. Custom tokens are saved in the addon preferences so they follow you across files and sessions, and they show up in the token sheets and builder next to the built-in tokens. (First release covers output paths; naming templates may follow later.)

    Phases¶

    • Define a custom token in the addon preferences: a name plus the scene data it reads
    • Live validity check + resolved-value preview as you define it (clear, specific error when it can't resolve)
    • Safe, sandboxed resolution — no way for a definition to crash Blender or run unrestricted code
    • Custom tokens appear in the token sheets / builder alongside built-ins (context-aware) and copy like any token
    • Persist custom tokens in the addon preferences (survive restart); sensible handling when a name clashes with a built-in token
    • Tests, in-app verification, changelog
